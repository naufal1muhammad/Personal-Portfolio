Info-Tech HRMS ChatBot

Served as the Project Lead in designing and developing a ChatBot algorithm to enhance user experience within the Human Resource Management System (HRMS) provided by my company. The ChatBot is integrated with a Python-based API, which serves as the knowledge base for answering user queries. This API is responsible for data retrieval, processing, and response generation and is developed using Python with Natural Language Toolkit (NLTK) and PyTorch as the core frameworks for natural language processing (NLP) and deep learning capabilities.

To ensure efficient environment management, Conda is used for handling software dependencies and creating isolated environments tailored for machine learning and NLP development. The implementation of NLTK allows the ChatBot to process human language efficiently, enabling accurate text analysis and intelligent responses.

As the Project Lead, I was responsible for overseeing the end-to-end development process, managing the technical implementation, and ensuring the seamless integration of the ChatBot within the HRMS. This project significantly improves HRMS user engagement by providing automated assistance, streamlining HR-related inquiries, and enhancing overall system usability.

KKDM V1.0

The mobile application is designed to display the company's PDA system as a web view without changes in the system's usage. The project's main objective is to maintain a constant connection and indicate to the user if connection is lost. The user can configure the HOST and PORT for the PDA system, and the application checks the connection status every ten seconds. Throughout the usage of the app, it displays the application version and connection status. If connection is lost, the application redirects to a page indicating a loss of connection with the inserted HOST and PORT, and includes a 'RETRY CONNECTION' button.

The app fetches a Base64 encoded string from the local server in the background, which contains the headquarters database HOST and PORT, and establishes a connection with it. If connection is lost, the app directs the flow to a page indicating a loss of connection with the database HOST and PORT, which includes a 'RETRY CONNECTION' button as well. The configuration page and connection info are password protected, and the user can access it at any given time during app usage.
